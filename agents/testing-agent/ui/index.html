Cypher 生成服务（CGS）

负责接收 `id + question`、向知识运营服务获取提示词、生成 Cypher、保存提示词快照并提交测试服务。

测试服务（Testing Service）

负责执行 TuGraph、进行规则评测与可选 LLM 复评，并在失败时创建问题票据。

知识修复建议服务（KRSS）

负责读取失败证据与提示词快照，生成知识修复建议并投递给知识运营服务。

知识运营服务（Knowledge Ops）

负责提供提示词包，并接收 KRSS 发出的知识修复建议以落地知识更新。

系统级联调 System Integration Run

系统会基于真实执行与评测结果自动判定是否进入知识修复闭环，直接观察问题如何穿过 CGS、Testing Service、KRSS 与 Knowledge Ops。
